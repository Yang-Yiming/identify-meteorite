# Mytest DINO Nearest-Neighbor Audit

Generated by:

```bash
python analysis/audit_mytest_neighbors.py \
  --out-dir analysis/mytest_audit_dino_v1 \
  --device cuda \
  --batch-size 32 \
  --num-workers 4 \
  --top-k 10
```

This audit uses frozen DINOv2 embeddings and computes, for each mytest image,
its nearest labeled/reference neighbors in `train + myval + test`.

## Outputs

- `manifest.csv`: combined manifest for train, myval, test, and mytest.
- `mytest_neighbor_summary.csv`: one row per mytest image with top-k source,
  label, and conflict metrics.
- `mytest_neighbors_topk.csv`: full top-k neighbor table.
- `summary.json`: compact JSON summary.

## What It Shows

- Many `mytest/rock` samples are nearest to train positive examples under DINO.
- Many `mytest/meteorite` samples are also closest to train positives, but a
  nontrivial subset are closest to train negatives.
- The mytest split is therefore not a clean source of “always safe” labels.

## Use

Use this file to identify:

- mytest samples that behave like label noise or domain outliers
- mytest samples that are very close to test-like reference regions
- candidate images for manual review or low-weight semi-supervised use
