# Test-Like Validation V1

Generated by:

```bash
python analysis/build_testlike_val.py
```

This is a first-pass proxy built without network-dependent DINO/CLIP features.
It uses lightweight image statistics only:

- RGB and saturation/value histograms
- grayscale thumbnail layout
- edge and texture-grid summaries

## Inputs

- train labels: `data/train_labels.csv`
- train crops: `preprocess/bbox_crop/train`
- myval labels: `data/myval/labels.csv`
- myval crops: `preprocess/bbox_crop/myval`
- test crops: `preprocess/bbox_crop/test`
- mytest raw images: `mytest/meteorite`, `mytest/rock`
- normal test anchor exclusion: `post_process/not-stone_best_071962.txt`

The test anchor contains 176 cropped test images. The force-zero not-stone IDs
are excluded from the target anchor distribution.

## Outputs

- `manifest.csv`: all discovered samples and paths.
- `test_like_scores.csv`: per-image test-likeness scores for train, myval,
  test, and mytest.
- `cluster_summary.csv`: cluster-level source and label composition.
- `test_like_val_top.csv`: 400 labeled train/myval samples with highest
  test-like score, balanced by label.
- `test_like_val_cluster.csv`: 400 labeled train/myval samples sampled from
  test-dense clusters, balanced by label and less concentrated than the top
  variant.
- `test_like_val_weighted.csv`: all labeled train/myval samples with a
  continuous `sample_weight` derived from test-likeness.
- `summary.json`: generation summary.

## Current Recommendation

Use `test_like_val_cluster.csv` as the first validation proxy candidate.

Reason:

- `test_like_val_top.csv` is more concentrated in a few clusters.
- `test_like_val_cluster.csv` covers more clusters and includes more myval
  positive samples while staying label-balanced.

Current split summary:

```text
test_like_val_cluster.csv:
  rows: 400
  labels: 198 negative, 202 positive
  sources: 346 train, 54 myval

test_like_val_top.csv:
  rows: 400
  labels: 200 negative, 200 positive
  sources: 364 train, 36 myval
```

## Caveats

This V1 feature space is intentionally crude. It can overemphasize color,
background remnants, image layout, and crop artifacts. It should not yet be
treated as ground truth. The next validation step is to score existing known
submissions/checkpoints on this proxy and verify whether it ranks mytest-heavy
runs below the current soup + reduced-not-stone SOTA.

The next feature upgrade should add frozen ConvNeXt penultimate features, then
DINO/CLIP/SigLIP embeddings if model weights are available.
