# Test-Like Validation DINO V3

Generated by:

```bash
python analysis/build_testlike_val.py \
  --out-dir analysis/testlike_dino_myval_v3 \
  --feature-backend dino_timm \
  --dino-model vit_base_patch14_dinov2 \
  --feature-sources myval,test \
  --fit-sources myval,test \
  --candidate-sources myval \
  --clusters 16 \
  --top-n 160 \
  --top-k 10 \
  --batch-size 32 \
  --num-workers 4 \
  --device cuda
```

This version uses frozen DINOv2 embeddings from `timm` instead of handcrafted
statistics. Only `myval` and `test` are embedded; only `myval` is eligible for
the validation candidates.

## Outputs

- `test_like_val_top.csv`: top 160 myval samples, balanced 80/80.
- `test_like_val_cluster.csv`: 160 myval samples from test-dense clusters,
  82 negatives / 78 positives.
- `test_like_val_weighted.csv`: all 332 masked myval samples with weights.

## Current Read

DINOv3 is much healthier than the handcrafted V1 proxy in absolute scale, but
it still does not cleanly rank the known bad mytest-heavy runs below the current
soup + reduced-not-stone result. Use it as a better diagnostic feature space,
not as the final selection rule.
