# Test-Like Validation Myval-Only V2

Generated by:

```bash
python analysis/build_testlike_val.py \
  --out-dir analysis/testlike_myval_v2 \
  --fit-sources myval,test \
  --candidate-sources myval \
  --clusters 16 \
  --top-n 160 \
  --top-k 10
```

This version is stricter than V1:

- train is not used to fit scaler/PCA/clusters/test-likeness.
- mytest is not used to fit scaler/PCA/clusters/test-likeness.
- candidate validation samples come only from myval.
- test anchor excludes `post_process/not-stone_best_071962.txt`.

The purpose is to remove the direct train/mytest contamination present in V1.
It does not guarantee that myval is clean or representative.

## Outputs

- `test_like_val_top.csv`: top 160 myval samples by test-likeness, balanced
  80 negative / 80 positive.
- `test_like_val_cluster.csv`: 160 myval samples sampled from test-dense
  clusters, currently 88 negative / 72 positive.
- `test_like_val_weighted.csv`: all 332 masked myval samples with continuous
  weights.

## Current Read

V2 produces more realistic absolute F1 values than V1, but it still does not
solve model selection. The mytest-augmented model remains high on this proxy,
which is consistent with the suspicion that myval itself is small, noisy, or
closer to mytest-style clean images than to Kaggle test.

Use this as a stricter diagnostic baseline, not as the final selection metric.
